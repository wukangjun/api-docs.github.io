    鑒權/認證

    請先生成測試網 或 主網API KEY。
    REST接口根URL:

    • 測試網:
      https://api-testnet.bybit.com

    • 主網(請根據您的網絡情況選擇以下任意一個地址使用):
      https://api.bybit.com
      https://api.bytick.com

    所有private接口都需要認證，public 接口不需要認證。

    公共參數

    需要簽名的接口必須包含以下參數:

    • api_key
    • timestamp - UTC毫秒時間戳
    • sign - 請求參數簽名

    另外我們提供可選的recv_window參數（單位是毫秒，默認值為5000），來指定請求在多長時間內有效，同時用來防止重放攻擊。

    註意，timestamp參數必須符合如下規則: server_time - recv_window <= timestamp < server_time + 1000，其中server_time是Bybit服務器時間，您可以通過服務器時間接口獲取。

    構建請求

    拼接參數示例（以獲取提現紀錄為例）

    param_str = "1660633147433T0d98KyVamQ62YBzN85000startTime=1659283200000&endTime=1660060800000&coin=USDT"

# timestamp: 1660633147433
# api_key: T0d98KyVamQ62YBzN8
# recv_window: 5000
# startTime=1659283200000
# endTime=1660060800000
# coin=USDT

    參數按照 字母順序 排列，然後計算sign。

    1. 時間戳 + api-key + [recv_window]+ [queryString(無需排序) | jsonBodyString]

    2. 使用HMAC_SHA256算法對第1步中拼接的string簽名，並轉換為16進製字符串，得出sign參數。

    GET請求格式（查詢提現紀錄）: 

    GET https://api-testnet.bybit.com/asset/v3/private/withdraw/record/query?startTime=1659283200000&endTime=1660060800000&coin=USDT HTTP/1.1
Host: api-testnet.bybit.com
-H 'X-BAPI-SIGN-TYPE: 2' \
-H 'X-BAPI-SIGN: eb431d99a1a203a434a82ac3ea8e107b5f94a967e9aaf922c41e84fb3ec9df78' \
-H 'X-BAPI-API-KEY: {api key}' \
-H 'X-BAPI-TIMESTAMP: 1658384431891' \
-H 'X-BAPI-RECV-WINDOW: 5000'

    POST請求格式（提現）:

    觸發劃轉動作

    要發送賬戶類型和父子不可知的轉賬，請使用創建萬能轉賬 接口。

    請註意該接口入參請求數據中的transferId字段為UUID全局唯一。

    請註意該接口支持期貨賬戶、現貨賬戶、理財賬戶。

    轉入和轉出賬戶類型不能相同。

    升級到統一保證金帳戶後，僅支持從現貨帳戶轉帳USDT到統保帳戶。

    HTTP 請求

    POST /asset/v3/private/transfer/inter-transfer

    請求參數

    參數 是否必須 類型 說明
    transferId true string UUID,全局唯一
    coin true string 幣種類型
    amount true string 兌入數量
    fromAccountType true string 賬戶類型
    toAccountType true string 賬戶類型

    返回參數

    參數 類型 說明
    transferId string UUID,全局唯一

    頻率限製

    有兩種類型的頻率限製: IP 頻率限製 (每秒计算) and API 頻率限製 (每分鐘计算). IP 頻率限制將計算您的所有請求，而 API 頻率限制僅計算您經過身份驗證的請求。

    IP頻率限製

    如果您收到 HTTP 403（拒絕訪問）響應, 您的 IP 已被暫時或永久禁止。 您應立即查看以下指南，以確保您的應用程序不會繼續違反限制。 如果您在 30 分鐘後仍被禁止，則您可能會被永久禁止。

    Bybit根據請求方式有不同的IP頻率限制。 我們不建議您在這些限制的邊緣運行您的應用程序，以防異常的網絡活動導致意外違規。

    • GET & DELETE 方法 (共享):
      • 連續 2 分鐘內每秒不超過 50 個請求
      • 連續 5 秒內每秒不超過 70 個請求
    • POST 方法:
      • 連續 2 分鐘內每秒不超過 20 個請求
      • 連續 5 秒內每秒不超過 50 個請求

    此表顯示了根據您使用的 API 的不同 IP 頻率限制數。

    IP 頻率限制 API Path
    2mins 50/s;
    5s 70/s
    统一保证金/unified
    反向永續/v2
    反向交割/futures
    USDT永續/public, /private
    賬戶資產/asset
    2mins 50/s;
    5s 70/s
    現貨v1、v3共享/spot/v1, /spot/v3
    2mins 50/s;
    5s 70/s
    USDC Options/option
    USDC Perpetual/perpetual

    違反限制後，您的 IP 將被禁止一段時間（通常為 30 分鐘）。 持續違反限制將導致永久禁止。 我們不能撤銷永久禁令或縮短臨時禁令。
